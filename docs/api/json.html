Chapter 1. Introduction

JavaScript Object Notation (JSON) is a type of data-interchange language:

JSON (JavaScript Object Notation) is a lightweight data-interchange format. It is easy for humans to read and write. It is easy for machines to parse and generate. It is based on a subset of the JavaScript Programming Language, Standard ECMA-262 3rd Edition - December 1999. JSON is a text format that is completely language independent but uses conventions that are familiar to programmers of the C-family of languages, including C, C++, C#, Java, JavaScript, Perl, Python, and many others. These properties make JSON an ideal data-interchange language.

- http://www.json.org/

You can obtain almost all information stored in Luntbuild in JSON format. JSON format is particularly useful in JavaScript because it is automatically recognized and parsed. Once parsed, the information will be available as any normal object structure with basic data types.

Chapter 2. Data Types

Chapter 3. Root Object

All JSON responces from Luntbuild have the same root (document) object, just like an XML document. The root object contains the System object, with all of the system wide information, and an array called result. The result array contains the objects returned by the query. The type of objects returned depends on the query. The root object may also have a member named error that contains any error messages encountered.

Root Data Members

NameType
systemobject (see System)
resultarray
errorstring

JSON text: 

{
	"system":{...},
	"result":[{...},...],
	"error":"Unsupported method"
}

Url to access luntbuild servlet


The servlet url should be specified correctly here in order to construct urls used in the notification message correct. Normally this value should be http://<server>:<port>/luntbuild/app, where <server> stands for your build server name or ip address, and <port> stands for the port number you used to access luntbuild. If this property is left empty, luntbuild will default this value to be "http://<server_ip>:8080/luntbuild/app", where <server_ip> stands for the ip address of the build server.

Working directory
You can optionally specify a working directory for the build system. Working directory is the directory where luntbuild used to hold checkouted artifacts from the version control system. If not specified, luntbuild will use the "work" directory under the luntbuild installation directory.
Publish directory
You can optionally specify a publish directory for the build system. Publish directory is the directory where luntbuild used to hold the output of the build process, thus the build log, and build artifacts, etc.
Set email related properties here if you want to notify user by email
SMTP host
You can optionally specify the SMTP mail host used by luntbuild to send email. If this property is not specified, luntbuild will use localhost as the default value.

post-build strategy
do not post-build: do not execute post-build script after build
post-build when success: execute post-build script only when build was successful
post-build when failed: execute post-build script only when build has failed
post-build always: always execute post-build script after build
notify strategy
Choose the notify strategy for this schedule. There are following strategies available:
notify when status changed: send notification when the status of the current build changed compared to the last build. That is, notification will be sent when the current build succeeds and the last build failed, or the current build fails and the last build succeeded.
notify when failed: send notification only when build failed
notify when success: send notification only when build succeeded
do not notify: do not send notification after build
notify always: always send notification after build

1.

How do you run different targets for a particular schedule of your project?

If you need to run different targets for a particular schedule of your project, it is good idea to name the schedule with the same name as the name of the target to be executed in your main build script file and use variable ${build.schedule.name} as a build target. This can be done for instance:

  1. Define your ant builder, and set the target to be ${build.schedule.name}, in this way, Luntbuild will call Ant with schedule name as targets.

  2. In your ant build file, add targets for different schedule names, ie. "nightly", "hourly", "release".

2.

How do you run multiple targets for a particular build of your project?

If you need to execute multiple targets (like > ant setup init dist package) just put them in your build file (for ant): 

            
            <target name="nightly" depends="setup init dist package"/>

3.

How to you define top location of your Ant target execution?

If you are using JNI or your application configuration depends on relative location of the files, you might have a problem running the application using Luntbuild. To resolve the issue, define env. variable with the top directory of your application, and use this variable to define location(s) in the configuration of your application. Then define the same environment variable in the appropriate Luntbuild builder.

4.

How do you create a new Project, Build, Schedule ...?

To create a new Project, Build, Schedule etc., look in upper right corner, just below the Tabs for an icon . The icon is quite small and for first time user it is hard to find. But as soon as you get used to look for it in all the tabs/pages, the small icon makes for nice and compact user interface.

5.

My favorite VCS doesn't work wit Luntbuild?

Make sure that external VCS application like cvs or p4 is in your default path when you are starting Luntbuild.

    Development FAQ

    1. Building Luntbuild in Eclipse
    1.

    Building Luntbuild in Eclipse

    The one problem I had with building Luntbuild source code using default Eclipse build is that starteam.jar is optional. This is checked in Ant <luntbuild>/build/build.xml and src. files that use StarTeam VCS are build conditionally based on the presence of starteam.jar. I have started building Luntbuild from cmd. line using ant, that works. But doing it directly from Eclipse is more convenient, so I use External Tools, I create Ant Build configuration, point it to <luntbuild>/build/build.xml and select clean package targets (make sure to put them in the right order). Then I use this External Tools target to re-build Luntbuild. If you want to use Eclipse default build facilities for Luntbuild project, here is another, possibly better option:

    Go to Luntbuild Eclipse project Properties

    Select Builders

    Create a new Builder (name it for example Luntbuild Builder)

    Select Ant Build, define workspace location, base directory, pick target "package"

    Save Luntbuild Builder, unselect Java Builder, and move Luntbuild Builder up

      Now you should be able to run Luntbuild build using std. Eclipse build facilities. Output from the is displayed in Console view.

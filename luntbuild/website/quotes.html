Web Luntbuild QuickBuild

"Continuous integration is a great step forward to better software project management. We were looking for a tool that meets the two following requirements : powerful enough to set up complex CI processes but easy to use on a daily basis. Amongst lots of existing tools, Luntbuild is the one that meets completely and perfectly these two requirements thanks to a user interface that is far more ergonomic than what can be seen on other tools. After some months of testing Luntbuild with 6 pilot projects, we will soon be in production mode with several tens of projects building on a single Luntbuild platform."

Actually, I hope that Luntbuild will be able to support the load of tens of projects! I've seen some feature requests about having a queue system in Luntbuild: IMHO, that would really help for supporting a big load.

As for my experience with Luntbuild, I have a little remark: When there are too many projects and too many schedules, the "Builds" page becomes difficult to read for the admin role. We have only 6 projects with 3 schedules each, which gives 18 lines on this first page, and when I log on as admin, it's not easy for me to find a specific schedule. Maybe that could be interesting to show the schedules in a tree (sorted by projects). It could be useful also to have a input field where you could type a
project or a schedule name, and with Javascript, that would hide any other project or schedule that doesn't match what you've typed in.

HTH!
Keep up the good work!

-- Fabrice BELLINGARD (Java architect at PSA Peugeot-Citroen)

The followings are some user comments at theServerSide news forum:

Luntbuild is very nice. I had set up CruiseControl in the past, and I was looking for something easier, and definitely found it in Luntbuild.

It's important to realize that Luntbuild is not just a continuous integration system. It's a full automated build and build management system. I've got our QA looking at using it for all of their builds (not scheduled, triggered manually) and for build management and promotion. It's a nice dashboard on your different project builds.

Some of the relatively unique features I like:

1) project build dependencies -> tell it which other builds this build depends on, then tell it if you'd like to build all dependencies before this build, build all projects which depend on this build after the build, both, or none

2) Accurev support -> ok, I built this, but none of the free ones I looked at supported it...

3) Build promotion -> so this QA build passed all regression tests and got approved by QA for release... now just go to that build in Luntbuild and promote it to the "gold release" project and you're done

All in all Luntbuild is very well done. The name's a little strange (ever thought of changing it?)... The UI's functional but a little clunky and not pretty, but that's probably just because they use Tapestry :-)

--Jason Carreira

 

We recently switched from cruisecontrol to luntbuild too, and find it much easier to configure because of the web interface, also the monitoring of builds is far easier for us with luntbuild.

We found with cruisecontrol that it took 5 minutes to build our code, but then half an hour to create an enormous xml log file. Also these logs were quickly filling up our hard disk.

I particularly like the luntbuild feature where you can tell it to keep the last x number of build results only.

Lastly, you can really see the robustness of this webapp, due I'm sure to its use of Hibernate, Spring and Tapestry.

--Mike Stephen

Web Luntbuild QuickBuild

  • Supports following Version Control Systems:
    • Accurev
    • Cvs
    • Visual Sourcesafe
    • Subversion
    • StarTeam
    • Perforce
    • Base Clearcase
    • Clearcase UCM
    • File system
    Build can be performed from one or more repositories of the same or different VCS type.
  • Following databases may be used to store Luntbuild data:
    • HSQLDB embedded (in process)
    • H2 embedded (in process)
    • H2 client/server
    • MySql
    • ProgreSql
    • SqlServer
    • Oracle
    • Derby client/server
  • Supports following build tools:
    • Ant
    •  Maven
    • Maven2
    • Shell script
    • Rake
  • ACL based Security (thanks Johannes Plachy).
  • Be able to setup dependencies between different projects.
  • Easy setup. All tasks are done using a clean web interface. No configuration files to edit!
  • History builds can be exactly re-produced.
  • Can perform clean build or incremental build.
  • Quiet period can be set before check out code for build
  • Build can be triggered in many ways, such as build manually, build when changes are detected in source repository, build after user checkin, or build when user-defined conditions are satisfied.
  • Build success condition can be defined by user, such as examine command return code, pattern match of command/build output, etc.
  • Source repository can get labeled after build to record exact source files making up of current build.
  •  Following notification methods can be used to send notificaion to specified users or checkin users after build completion:
    • Email
    • MSN Messenger
    • Jabber Messenger
    • Sametime
    • Blog
  •  Build artifacts might be published on the web to provide a central download area for the whole team.
  • You can add patch for a particular build using file uploading.
  • Builds are categorized, and they can get promoted.
  • Build can be searched and deleted.
  • Build log, revision log and system log4j log can be directly accessed by web interface.
  • Different version of JDK and JDK options, different versions of Ant, and Ant options can be configured for different project/views.
  • Schedule can be periodically or cron like.
  • API available to integrate your applications with Luntbuild.
  • Flexible versioning strategy, such as automatic increases of user defined version number. To be able to embed dynamic fields such as build date, build iterations etc. into build version.
  • Be able to stop running build.
  • Data can be exported to XML file or imported from an XML file.
  • Display summary of [http://junit.org JUnit] test results on build page
  • The ability to clone/copy a project
  • Be able to configure "build cleanup strategy" which gives user
    the option to only keep a specified number of builds, or to keep
    specified build days for particular schedules
